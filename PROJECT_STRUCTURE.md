# Project Structure

This project is a Naver News Crawler designed to collect news articles from Naver News. It allows users to specify search queries, date ranges, and other filters to gather news URLs and subsequently extract their content.


## Python Files

This section describes the purpose and functionality of each Python script in the project.

### `main.py`

*   **Purpose**: This is the main entry point of the application.
*   **Functionality**: 
    *   Parses command-line arguments (or uses interactive mode via `user_interface.py`).
    *   Initializes and coordinates the news crawling process based on user-defined options.
    *   Can delegate to `daily_collector.py` if date-range based collection is specified.
    *   Uses `url_extractor.py` to fetch news URLs and `news_content_extractor.py` to extract article content.

### `search_options.py`

*   **Purpose**: Defines how search URLs are constructed for Naver News.
*   **Functionality**:
    *   Contains the `NaverNewsSearchOption` class.
    *   Allows setting various search parameters like query keyword, sort order (relevance, recent, oldest), search period (all, 1h, 1d, custom date range, etc.), and news type (all, photo, video, etc.).
    *   Builds the final Naver News search URL based on the configured options.

### `url_extractor.py`

*   **Purpose**: Extracts news article URLs from Naver News search result pages.
*   **Functionality**:
    *   Contains the `NaverNewsURLExtractor` class.
    *   Takes a Naver News search URL (typically generated by `search_options.py`).
    *   Handles pagination in search results (infinite scroll).
    *   Extracts both Naver News article URLs (e.g., `n.news.naver.com/...`) and original press company article URLs.
    *   Filters duplicate URLs.
    *   Saves the collected list of URLs to a JSON file, including metadata like the search query, period, and collection timestamp.

### `news_content_extractor.py`

*   **Purpose**: Fetches and extracts content from individual news article URLs.
*   **Functionality**:
    *   Contains the `NaverNewsContentExtractor` class.
    *   Loads a list of URLs from a file (typically generated by `url_extractor.py`).
    *   For each URL, fetches the HTML content of the news article.
    *   Parses the HTML to extract information such as title, press company, publication date, main article body, and reporter's name.
    *   Handles potential errors during fetching and parsing.
    *   Saves the extracted news content into JSON files. It supports batching, where multiple articles (e.g., up to 20) are saved in a single file, with filenames indicating the query, period, and batch range.
    *   Supports different extraction modes (`sequential`, `balanced`, `per_date`) using `balanced_extractor.py` to select which URLs to process if a limit is set.

### `daily_collector.py`

*   **Purpose**: Manages the collection of news articles over a specified range of dates.
*   **Functionality**:
    *   Contains the `NaverNewsDailyCollector` class.
    *   Takes a start date, end date, and other crawling options.
    *   For each day in the specified range, it invokes the main crawling logic (similar to `main.py`'s `run_crawler` function) to collect news for that specific day.
    *   Supports parallel execution using `ThreadPoolExecutor` to speed up the collection process for multiple days.
    *   Aggregates the results (URLs and content) from all processed days into final output files.
    *   Manages temporary directories for daily outputs and cleans them up.
    *   Saves overall collection statistics.

### `balanced_extractor.py`

*   **Purpose**: Provides utility functions for selecting a balanced subset of news URLs, typically based on date distribution.
*   **Functionality**:
    *   `group_urls_by_date()`: Groups a list of URL items by their 'search_date'.
    *   `extract_balanced_urls()`: Selects URLs from the grouped list. It supports:
        *   `sequential`: Takes URLs from the beginning of the list.
        *   `balanced`: Distributes the total limit of URLs as evenly as possible across available dates.
        *   `per_date`: Takes a specified number of URLs from each date group.

### `user_interface.py`

*   **Purpose**: Provides an interactive command-line interface for users to configure crawling options.
*   **Functionality**:
    *   Contains the `NaverNewsUserInterface` class.
    *   Presents a series of prompts to the user to set options like search query, period, sorting, content extraction preferences, delays, etc.
    *   Validates user input for certain fields (e.g., date formats, numerical inputs).
    *   Returns a dictionary of options that can be used by `main.py` to start the crawling process.

### `test_improvements.py`

*   **Purpose**: Contains test functions for specific features or improvements.
*   **Functionality**:
    *   Includes tests for features like press filtering (`test_press_filter`) and duplicate news detection (`test_duplicate_news_detection`).
    *   Seems to be used for development and verification of new or modified functionalities rather than being part of the main crawling workflow.

## Overall Workflow

The project generally follows this workflow:

1.  **Configuration**: 
    *   The user runs `main.py`.
    *   If the `--interactive` (or `-i`) flag is used, `user_interface.py` is invoked to guide the user through setting various crawling options (search query, date range, extraction preferences, etc.).
    *   Otherwise, options are taken from command-line arguments parsed by `main.py`.

2.  **Search URL Generation**: 
    *   Based on the configured options, `search_options.py` (specifically the `NaverNewsSearchOption` class) constructs the appropriate Naver News search URL.

3.  **URL Collection**: 
    *   `url_extractor.py` (specifically `NaverNewsURLExtractor`) takes the search URL.
    *   It navigates through the search result pages (handling pagination) and collects news article URLs.
    *   It can distinguish between Naver-hosted news URLs and URLs of the original articles on press company websites.
    *   The collected URLs, along with metadata (query, period, timestamp), are saved into a JSON file in the directory specified by `--url-output` (default: `url_data`).

4.  **Content Extraction (if enabled)**:
    *   If the `--extract-content` option is enabled, `news_content_extractor.py` (specifically `NaverNewsContentExtractor`) is used.
    *   It reads a URL file (generated in the previous step).
    *   For each URL (or a selected subset if limits/balancing are applied via `balanced_extractor.py`), it fetches the article's webpage.
    *   It parses the HTML to extract details like the title, press name, publication date, main content, and reporter info.
    *   The extracted content is saved in batches to JSON files in the directory specified by `--output` (default: `news_data`).

5.  **Daily Collection Mode (if enabled)**:
    *   If the `--daily-collect` option is used (typically with a custom date range), `main.py` delegates the task to `daily_collector.py` (`NaverNewsDailyCollector`).
    *   The `daily_collector` iterates through each day in the specified date range.
    *   For each day, it essentially runs an independent crawling process (steps 2-4) to gather news specific to that day.
    *   It can perform these daily collections in parallel using multiple threads to improve efficiency.
    *   After processing all days, it aggregates the collected URLs and extracted content into consolidated output files.
    *   It also manages temporary storage for daily results and cleans them up.

6.  **Output**: 
    *   The primary outputs are JSON files containing lists of collected URLs and extracted news content, stored in the respective output directories.
    *   Log files provide information about the crawling process.
    *   In daily collection mode, a statistics file summarizing the collection across the date range is also produced.
