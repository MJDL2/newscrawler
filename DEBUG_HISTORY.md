# 네이버 뉴스 크롤러 디버그 역사

## 개요

이 문서는 네이버 뉴스 크롤러 개발 과정에서 발생한 주요 버그와 해결 과정을 기록합니다.

## 2025-05-23: 병렬 처리 구현 및 안정화

### 이슈 1: ThreadPoolExecutor 도입 시 진행상황 표시 문제
**심각도**: 중간  
**발생일**: 2025-05-23  

**문제 상황**:
- 병렬 처리 도입 후 개별 작업의 진행상황을 실시간으로 추적하기 어려움
- tqdm 프로그레스 바가 병렬 작업과 제대로 동기화되지 않음

**해결 방법**:
- ThreadPoolExecutor와 tqdm의 올바른 연동
- 순서 보장을 위한 인덱스 기반 결과 저장
- 예외 처리에서도 진행률 업데이트 보장

**예방 조치**:
- 병렬 작업 시 순서 보장을 위한 인덱스 기반 결과 저장
- 예외 처리에서도 진행률 업데이트 보장

## 2025-05-21: 사용자 인터페이스 개선 및 버그 수정

### 이슈 1: 날짜별 수집 기능 동작 불일치
**심각도**: 높음  
**발생일**: 2025-05-21  

**문제 상황**:
- 사용자 인터페이스에서 1주일, 1개월 등 기간에서도 '날짜별 수집' 옵션이 표시됨
- 실제 실행 시 오류 발생

**해결 방법**:
- UI 로직과 실제 처리 로직 간 일관성 확보
- custom 기간에서만 날짜별 수집 옵션 제공

**학습 내용**:
- UI 로직과 실제 처리 로직 간 일관성 유지 필요
- 사용자에게 제공되는 옵션은 실제 지원 가능한 기능과 정확히 일치해야 함

### 이슈 2: URL 수집 제한이 날짜별 수집에 미적용
**심각도**: 중간  
**발생일**: 2025-05-21  

**문제 상황**:
- 사용자 인터페이스에서 'URL 수집 제한'을 설정해도 날짜별 수집에 반영되지 않음

**해결 방법**:
- main.py에서 daily_collector에 모든 설정값 올바른 전달
- 옵션 누락 방지를 위한 체계적 전달 구조

**예방 조치**:
- 새로운 옵션 추가 시 모든 경로에서 전달되는지 확인
- 통합 설정 관리 클래스 도입 검토

### 이슈 3: 제목 유사도 기반 중복 필터링 성능 문제
**심각도**: 낮음  
**발생일**: 2025-05-21  

**문제 상황**:
- 대량 URL 수집 시 중복 필터링 과정에서 성능 저하
- O(n²) 복잡도로 인한 처리 시간 증가

**해결 방법**:
- 제목 길이 기반 사전 필터링
- 해시 기반 빠른 중복 체크
- 유사도 계산 최적화

**학습 내용**:
- 알고리즘 복잡도 고려한 구현 필요
- 대용량 데이터 처리 시 성능 테스트 중요
