# 네이버 뉴스 크롤러 개발 목표 및 진행상황

## 프로젝트 목표

### 핵심 목표
1. **안정적인 뉴스 데이터 수집**: 네이버 뉴스 검색을 통한 대량 뉴스 데이터 자동 수집
2. **효율적인 처리 성능**: 병렬 처리 및 최적화를 통한 수집 시간 단축
3. **사용자 친화적 인터페이스**: CLI와 대화형 인터페이스를 통한 직관적 사용
4. **확장 가능한 아키텍처**: 모듈화된 구조로 기능 확장 용이성 확보

### 세부 기능 목표
- 검색어 기반 뉴스 URL 수집
- 언론사별 본문 추출 및 저장
- 날짜별/기간별 수집 지원
- 중복 기사 필터링
- 다양한 추출 모드 제공
- 실시간 진행상황 표시

## 현재 진행상황 (2025-05-23 기준)

### ✅ 완료된 기능

#### 1. 병렬 처리 시스템 (2025-05-23)
- **ThreadPoolExecutor 도입**: 여러 날짜 동시 처리
- **명령행 인자 확장**: `--max-workers` 옵션 추가
- **사용자 인터페이스 확장**: 대화형 모드에서 병렬 처리 설정
- **추출 모드 옵션 반영**: 날짜별 수집에서 extraction_mode 지원

#### 2. 핵심 크롤링 기능 (2025-05-21)
- **검색 및 URL 수집**: 네이버 뉴스 검색 결과에서 URL 추출
- **본문 추출**: 수집된 URL에서 뉴스 본문 텍스트 추출
- **날짜별 수집**: 지정된 날짜 범위를 일별로 나누어 수집
- **JSON 저장**: 수집된 데이터를 구조화된 JSON 형태로 저장

#### 3. 사용자 경험 개선 (2025-05-21)
- **프로그레스 바**: tqdm 라이브러리를 활용한 진행상황 시각화
- **대화형 인터페이스**: 단계별 안내와 옵션 설정
- **로그 출력 개선**: 필요한 정보만 표시하는 간소화된 로깅

#### 4. 데이터 품질 향상 (2025-05-21)
- **중복 기사 필터링**: 제목 유사도 기반 중복 감지 및 제거
- **언론사 필터링**: 네이버 검색 옵션을 활용한 언론사 필터링
- **균등 추출**: 날짜별로 고르게 기사를 추출하는 모드 추가

#### 5. 설정 및 옵션 체계 (2025-05-21)
- **URL/본문 수집 제한**: 수집량 제어를 위한 명확한 옵션 체계
- **추출 모드**: sequential, balanced, per_date 방식 지원
- **검색 옵션**: 기간, 정렬, 언론사 등 다양한 검색 조건 설정
