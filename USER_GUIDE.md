## 1. 설치 방법

### 1.1 필수 요구사항

- Python 3.7 이상
- pip (Python 패키지 관리자)
- Git (선택 사항, 코드 복제용)

### 1.2 저장소 복제

```bash
git clone https://github.com/your-username/naver-news-crawler.git
cd naver-news-crawler
```

또는 소스 코드를 직접 다운로드하여 압축을 풀고 해당 디렉토리로 이동합니다.

### 1.3 필수 패키지 설치

```bash
pip install -r requirements.txt
```

주요 필수 패키지:
- requests: 웹 요청 처리
- beautifulsoup4: HTML 파싱
- tqdm: 진행 상황 표시 (선택사항)

## 2. 기본 사용법

### 2.1 명령행 실행

기본적인 뉴스 크롤링은 다음 명령으로 실행합니다:

```bash
python main.py "검색어"
```

추가 옵션:

```bash
python main.py "검색어" --period 1w --extract-content --content-limit 50
```

주요 옵션:
- `--period`: 검색 기간 (all, 1h, 1d, 1w, 1m, 3m, 6m, 1y, custom)
- `--extract-content`: 뉴스 본문 추출 활성화
- `--content-limit`: 추출할 본문 개수 제한
- `--max-urls`: 수집할 URL 개수 제한
- `--help`: 전체 옵션 목록 확인

### 2.2 출력 파일

크롤링 결과는 다음 위치에 저장됩니다:
- URL 목록: `url_data/` 디렉토리
- 뉴스 본문: `news_data/` 디렉토리

## 3. 대화형 인터페이스 사용법

더 쉬운 사용을 위해 대화형 인터페이스를 제공합니다:

```bash
python main.py -i
```

대화형 인터페이스는 단계별로 필요한 설정을 안내합니다.

### 3.1 날짜별 수집 사용 방법

**주의: 날짜별 수집 기능은 기간을 직접 지정(custom)한 경우에만 사용할 수 있습니다.**

날짜별 수집을 위한 단계:
1. 검색 기간 설정에서 "9. 직접 날짜 입력 (custom)"을 선택합니다.
2. 시작일과 종료일을 YYYYMMDD 형식으로 입력합니다.
3. "해당 기간을 일별로 나누어 각각 수집하시겠습니까?" 질문에 "Y"로 답합니다.

다른 기간 설정(1주일, 1개월 등)에서는 날짜별 수집이 지원되지 않습니다.## 4. URL 수집과 본문 추출 이해하기

네이버 뉴스 크롤러는 2단계로 작업이 진행됩니다:

```
┌─ 검색 결과 ─────────────┐      ┌─ 수집된 URL ──────────┐      ┌─ 추출된 본문 ─────────┐
│ URL 1                    │      │ URL 1                 │      │ 본문 1                │
│ URL 2                    │      │ URL 2                 │      │ 본문 2                │
│ URL 3                    │ ──→  │ URL 3                 │ ──→  │ 본문 3                │
│ ...                      │      │ ...                   │      │ ...                  │
│ URL 1000+                │      │ URL 최대 [입력값]개    │      │ 본문 최대 [입력값]개   │
└──────────────────────────┘      └─────────────────────────┘      └─────────────────────────┘
   <네이버 검색 결과>                <URL 수집 단계>                 <본문 추출 단계>
```

### 4.1 URL 수집 단계

1단계에서는 네이버 검색 결과에서 뉴스 URL을 수집합니다.

- **대화형 메뉴**: "URL 수집 제한 설정"
- **명령행 옵션**: `--max-urls`
- **의미**: 네이버 검색에서 몇 개의 URL을 수집할지 설정
- **값 설정**:
  - `0`: 제한 없음 (검색 결과의 모든 URL 수집)
  - `양수(예: 100)`: 지정된 개수까지만 URL 수집

### 4.2 본문 추출 단계

2단계에서는, 1단계에서 수집한 URL에서 실제 뉴스 기사 본문을 추출합니다.

- **대화형 메뉴**: "본문 추출 제한 설정"
- **명령행 옵션**: `--content-limit`
- **의미**: 수집된 URL 중 몇 개의 본문을 추출할지 설정
- **값 설정**:
  - `0`: 제한 없음 (수집된 모든 URL에서 본문 추출)
  - `양수(예: 50)`: 지정된 개수까지만 본문 추출

### 4.3 두 단계의 관계

- **중요**: URL 수집 개수가 본문 추출 개수보다 적으면, 최대 URL 수집 개수까지만 본문이 추출됩니다.
  - 예: URL 수집=50, 본문 추출=100 설정 시 → 최대 50개 본문만 추출됨

### 4.4 본문 추출 방식

본문 추출 제한을 설정할 때, 어떤 URL의 본문을 추출할지 선택하는 방식을 지정할 수 있습니다:

- **sequential** (순차적): 목록 앞에서부터 순서대로 추출
- **balanced** (균등 분포): 전체 범위에서 균등하게 분포된 URL 선택
- **per_date** (날짜별 분포): 날짜별로 균등하게 분포된 URL 선택

### 4.5 설정 예시

다음은 몇 가지 일반적인 설정 예시입니다:

| 원하는 결과 | URL 수집 설정 | 본문 추출 설정 | 추출 방식 |
|------------|--------------|--------------|----------|
| 모든 기사 완전 수집 | 0 (무제한) | 0 (무제한) | - |
| 처음 100개 URL만 수집, 모두 본문 추출 | 100 | 0 (무제한) | - |
| 모든 URL 수집, 첫 50개만 본문 추출 | 0 (무제한) | 50 | sequential |
| 500개 URL 수집, 균등하게 100개 본문 추출 | 500 | 100 | balanced |
| 모든 URL 수집, 날짜별로 50개씩 본문 추출 | 0 (무제한) | 50 | per_date |

## 5. 자주 묻는 질문

### 5.1 크롤링 속도를 높일 수 있나요?

속도는 `--delay`와 `--content-delay` 옵션으로 조정할 수 있습니다. 그러나 너무 짧게 설정하면 네이버에서 차단될 수 있으니 주의하세요.

### 5.2 특정 언론사만 수집할 수 있나요?

현재 버전에서는 간접적으로만 가능합니다. 향후 업데이트에서 직접적인 언론사 필터링 기능이 추가될 예정입니다.

### 5.3 크롤링이 중간에 끊겼습니다. 이어서 할 수 있나요?

아직 자동 복구 기능은 없지만, 수집된 URL 파일을 이용해 본문 추출만 다시 실행할 수 있습니다.

## 6. 문제 해결

### 6.1 'module not found' 에러

필수 패키지가 누락되었습니다. 다음 명령을 실행하세요:
```bash
pip install -r requirements.txt
```

### 6.2 빈 결과가 나옵니다

검색어나 기간 설정을 확인하세요. 해당 검색어로 검색 결과가 없을 수 있습니다.

### 6.3 본문이 제대로 추출되지 않습니다

일부 언론사의 경우 HTML 구조가 변경되어 추출이 안 될 수 있습니다. 이슈를 제보해 주시면 업데이트하겠습니다.
